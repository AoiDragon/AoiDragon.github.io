算法学习笔记——扩展欧几里得算法

算法笔记 扩展欧几里得算法
算法学习笔记
发布日期:   2020-02-07

一、辗转相除法（欧几里得算法）

辗转相除法是计算最大公约数的算法，也是扩展欧几里得算法的基础。

设gcd(a, b)是计算自然数a和b的最大公约数的函数。令a=p*b+q。观察到a可以拆成两项，一项含b，另一项含q。所以gcd(b, p)整除a。显然gcd(b, p)整除b，所以gcd(b, p)整除gcb(a, b)。且q=a-p*b。由类似思路，gcb(a, b)整除q和b，也因此整除gcd(b, p)。所以gcd(a, b)=gcd(b, p)=gcd(b, a%b)。

继续这一过程，最终会得到gcd(a,b)=gcd(c, 0)=c。

二、扩展欧几里得算法

介绍

扩展欧几里得算法是在已知a、b的情况下，求解使得ax+by=gcd(a, b)的整数x、y。

我们令G=gcd(a, b)。现在已知ax+by=G。我们仿照辗转相除法，假设有x1、y1使bx1+(a mod b)y1=G。所以有ax+by=bx1+(a mod b)y1。而a mod b 可以表示为a-b(a/b) 。所以该方程有可化为ax+by=ay1+b(x1−(a/b)y1)，一定有特解x=y1,y=x1−(a/b)y1，这样就找到了递推关系。

由于在替换方式和辗转相除一致，我们最后一定会得到一组a、b，a=G, b=0。所以Gx+0*y=G,只需要令x=1即可。（尽管y的值对结果无影响，但还是建议设为0，防止递归中数据不断累加导致越界）

例题

我们以这一题为例P1082-同余方程:求关于x 的同余方程ax ≡ 1 (modb) 的最小正整数解。

以下内容是我对学委大佬的题解的个人理解。

问题转化

问题的实质是求满足ax+by=1的最小的x。但扩展欧几里得求解的是满足ax+by=gcd(a, b)的整数x、y。这二者是怎么联系在一起的呢？这是因为ax+by=m有解的充分必要条件是m mod gcd(a,b)=0。因为m可以看作由含a的因式和含b的因式组成，而a和b又都可以被gcd(a, b)整除，所以m一定可以被gcd(a, b)整除。

而本题中m=1，所以gcd(a, b)只能等于1。这样问题就转化为扩展欧几里得算法的题目了。

答案处理

还有一点小问题。我们求出来的确实满足等式的条件，但并不能保证是满足条件的最小正整数。为了解决这一问题，我们可以给x减去或加上一定数量的b来解决。这是因为原等式可化为y=(1-ax)/b，我们只需要确保y在改动后仍然是整数即可。

完成这一操作我们只需要一个表达式：x = (x % b + b) % b;

对于大于b的x，x%b这一操作就相当于批量减去b；而对于小于b的x，x%b相当于批量加b直到x>b再减b。

但在C++中，(-5)%3的结果是-2而非1。其他类似的表达式也会得出余数是负数的结果。所以我们还需要再做进一步处理，将结果加b再对b取余即可。

代码

#include<iostream>

using namespace std;

long long int a, b, x, y;

void exgcd(long long int a, long long int b)
{
    if(b==0)
    {
        x=1;
        y=0;
        return;
    }
    //传递下一层的系数，先求解下一层的方程
    exgcd(b, a%b);
    long long int temp =x;
    //此时已经获得了下一层的x和y的值，开始根据递推公式更新本层的x和y
    x=y;
    y=temp-(a/b)*y;
    return;
}

int main(void)
{
    cin>>a>>b;
    exgcd(a, b);
    x=(x%b+b)%b;
    cout<<x<<endl;
    return 0;
}
