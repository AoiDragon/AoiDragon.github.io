《挑战程序设计竞赛》———Saruman's Army(POJ 3069)

贪心
ACM题解
发布日期:   2020-02-01

题目描述

vjudge上的原题在这里

题目描述是建立再《魔戒》的基础上的，读起来还蛮有意思的。

大意是白袍萨鲁曼老爷子要带领军队从大本营Isengard出发前往攻打洛汗的圣盔谷。为了掌握军队的动向，他把若干个水晶球放置在了队伍中。水晶球有一定的观察半径，且必须由一支部队携带。现在请你算出萨鲁曼老爷子最少需要多少水晶球才能监控到他所有的部队。

输入

本题输入包括多个用例。

对于每个用例，第一行为水晶球的最大观察半径R(0<=R<=1000)和萨鲁曼部队数量n(1<=n<=1000)。第二行为每支部队所在的位置x1,…,xn。当R=n=-1时，结束输入。

输出

对每个样例，输出所需水晶球的最小个数

输入样例

0 3
10 20 20
10 7
70 30 1 7 15 20 50
-1 -1

输出样例

2
4

个人题解

本题的基本思路是贪心。开始操作前先对所给位置进行排序方便操作。

首先从左侧第一点开始考虑。这一点必须处在监视半径内，所以我们可以先将这个点的坐标加上半径值，得出水晶球最远可以放在哪里。题目的要求是水晶球的最少数目，所以一个水晶球自然是覆盖越多部队越好。所以我们从左侧的点开始对每一个点进行判断，找出最远点左侧最近的部队位置，也就是水晶球摆放的最佳位置（称为标记点）。然后我们再以第一个标记点的坐标加上半径，得出它右侧的覆盖范围，这范围内的所有点就不再需要考虑了。而在范围外的第一个点的情况是不是就和我们一开始左侧第一点的情况完全一样了呢。所以接下来重复上面的操作即可。

总结下来就是两步。

  1. 从起点开始，通过半径判断标志点的最佳摆放位置。
  2. 从标记点开始，通过半径判断下一个起点的位置。

每一个循环后都能找到一个标记点，递增答案即可。

思路其实并不复杂，但还有几个细节需要注意。

  1. 如何判断所有的点都已被纳入考虑。

    我一开始是判断当前的位置有没有超过输入中最远点的位置。但输入样例中的第一个数据就会产生问题。当遇到最远点不止一个的情况下会产生遗漏。最后我选择用flag记录已经判断过的点的数量，再与n比较即可。

  2. 格式

    这个纯粹是个人的粗心大意（或者说刷题不够）。比如由于有多组样例，ans和flag在每一次循环后要归零。还有要仔细读题，看清楚变量的输入顺序。我提交后wa了一次就是因为循环内的读取顺序错了导致只能读取一次输入。。。

下面是我的代码。

#include <iostream>
#include <algorithm>

using namespace std;

int n, R, max, ans, j, flag;
int x[1050];

int main(void)
{
    cin>>R>>n;
    while(n!=-1&&R!=-1)
    {
        for(int i=0; i<n; i++)
            cin>>x[i];
        sort(x, x+n);
        int mile=x[0];
        int num=0;
        while(flag!=n)
        {
            mile+=R;
            for(j=num; j<n; j++)
            {
                if(x[j]>mile)
                    break;
                else
                    flag++;
            }
            ans++;
            mile=x[j-1]+R;
            num=j;
            for(j=num; j<n; j++)
            {
                if(x[j]>mile)
                    break;
                else
                    flag++;
            }
            mile=x[j]; 
            num=j;   
        }
        cout>>ans>>endl;
        ans=0;
        flag=0;
        cin<<R<<n;
    }
    return 0;
}

书本题解

书本上的题解要简略许多，发上来学习一下（这里只是处理一次输入的版本）。

#include <iostream>
#include <algorithm>

using namespace std;

int N, R;
int x[1050];

int main(void)
{
    cin>>R>>N;
    for(int i=0; i<N; i++)
        cin>>x[i];
    int i=0, ans=0;
    while(i<N)
    {
        int s=x[i];//起点
        while(x[i]<=s+R&&i<N)
            i++;
        int p=x[i-1];//标记点
        while(x[i]<=p+R&&i<n)
            i++;
        ans++;
    }
    cout<<ans<<endl;
    return 0;  
}

书上的题解省去了我自己题解中许多不必要的变量。比如直接用i来判断是否全部处理、用while循环来寻找起点和标记点等，值得我学习。
