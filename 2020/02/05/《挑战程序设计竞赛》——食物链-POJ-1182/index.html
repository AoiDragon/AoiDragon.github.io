《挑战程序设计竞赛》——食物链(POJ 1182)

并查集 《挑战程序设计竞赛》
ACM题解
发布日期:   2020-02-05

题目

原题链接

题目描述

动物王国中有三类动物A,B,C，这三类动物的食物链构成了有趣的环形。

A吃B， B吃C，C吃A。

现有N个动物，以1－N编号。

每个动物都是A,B,C中的一种，但是我们并不知道它到底是哪一种。

有人用两种说法对这N个动物所构成的食物链关系进行描述：

第一种说法是”1 X Y”，表示X和Y是同类。

第二种说法是”2 X Y”，表示X吃Y。

此人对N个动物，用上述两种说法，一句接一句地说出K句话，这K句话有的是真的，有的是假的。

当一句话满足下列三条之一时，这句话就是假话，否则就是真话。

  1. 当前的话与前面的某些真的话冲突，就是假话；
  2. 当前的话中X或Y比N大，就是假话；
  3. 当前的话表示X吃X，就是假话。

你的任务是根据给定的N和K句话，输出假话的总数。

输入格式

第一行是两个整数N和K，以一个空格分隔。

以下K行每行是三个正整数 D，X，Y，两数之间用一个空格隔开，其中D表示说法的种类。

若D=1，则表示X和Y是同类。

若D=2，则表示X吃Y。

输出格式

只有一个整数，表示假话的数目。

数据范围

1≤N≤50000,
0≤K≤100000

输入样例

100 7
1 101 1 
2 1 2
2 2 3 
2 3 3 
1 1 3 
2 3 1 
1 5 5

输出样例

3

思路1——种类并查集

如果仅仅涉及对不同种的动物进行分类，普通的并查集就足够了。但本题中还涉及了不同种类动物间的捕食关系。

所以我们要对并查集做一定的修改。对于每只动物，我们创建3个元素i-A,i-B,i-C而不是只创建1个元素，并用这3*N个元素建立并查集。其中：

  1. i-x表示“i属于种类x”。
  2. 并查集里的每一个组表示组内元素代表的情况都同时发生或不发生。

所以对于每一条信息我们只需要做如下操作：

  1. 若x和y属于同一种类：合并x-A和y-A、x-B和y-B、x-C和y-C。
  2. 若x捕食y：合并x-A和y-B、x-B和y-C、x-C和y-A。

仅凭一组输入无法判断动物的种类，那我们索性三种情况都考虑，再判断是否矛盾即可。

#include<iostream>
#include<algorithm>
#include<cstdio>

using namespace std;

int N, M, d, x, y, ans;
int par[150050];

int find (int n)
{
    if(par[n]==n)
        return n;
    else
        return par[n]=find(par[n]);
}

void unite(int x, int y)
{
    //合并操作一定是从两颗树的根节点处合并
    //fx是根节点，不是父节点！！
    int fx=find(x);
    int fy=find(y);
    par[fx]=fy;
    return;
}

bool same(int x, int y)
{
    return find(x)==find(y);
}

int main(void)
{
    cin>>N>>M;
    for(int i=0; i<=3*N; i++)
        par[i]=i;        
    for(int i=0; i<M; i++)
    {
        cin>>d>>x>>y;
        //排除错误输入
        if(x>N||y>N||x<=0||y<=0)
        {
            ans++;
            continue;
        }
        //x和y同类
        if(d==1)
        {
            //判断二者之前是否存在捕食关系
            if(same(x, y+N)||same(x, y+2*N))
                ans++;
            //先判断，再合并，相当于舍弃矛盾数据
            else
            {
                //此处也可理解为x的同类是y的同类，x的猎物是y的猎物，x的天敌是y的天敌
                unite(x, y);
                unite(x+N, y+N);
                unite(x+2*N, y+2*N);
            }
        }
        //x捕食y
        else
        {
            //排除x捕食自己、x和y同种、x被y捕食的情况
            if(x==y||same(x, y)||same(x, y+2*N))
                ans++;
            else
            {
                unite(x, y+N);
                unite(x+N, y+2*N);
                unite(x+2*N, y);
            }                            
        }
    }
    cout<<ans<<endl;
    return 0;
}

思路2——带权并查集

第一种方法相当于用了3个并查集，有没有占用空间更小的办法呢？仔细思考一下，如果我们对每个节点再赋予一个值来表示它当前的状态怎么样呢？带权并查集大概就是做了这样的改动。

权值

我们把这样的值称为权。在对并查集进行路径压缩和合并操作时，这些权值具有一定属性，即可将他们与父节点的关系，变化为与所在树的根结点关系。

也就是说，权值代表着当前节点与父节点的某种关系（即使路径压缩了也是这样），通过两者关系，也可以将同一棵树下两个节点的关系表示出来。

在如HDU-3038这样的题目中，权值是不同节点间的数值差。而对于本题，每一只动物和它的根节点间的关系只有3种：0–同类、1–捕食、2–被捕食。那么我们只需要在计算权值时对3取余即可。

例如：现在有这样一条食物链：A->B->C->D->E，想要求出A关于根节点E的权值，我们只需将从A到E之间每个节点到其父节点对应的权值相加再对3取余（因为三种动物的捕食关系成环状，每隔三个状态就会循环一次）。

最后注意一开始把权值都设为0（自己与自己是同类）

权值转移

如何在并查集的两个基本操作合并和查找中转移权值？

合并

参考下图，x、px属于一棵树，y、py属于另一棵，通过输入我们知道了x相对于y的权值。我们要把A树合并的到B树中，就需要确定px相对于py的权值。

把权值看作向量一样的有向线段可以帮助我们理解。如下图，由向量的加法，易得w[px]=w[b]-w[a]+s。

最后再处理一下细节。w[b]-w[a]可能小于0，所以我们先对其加3再取模。由题意s只可能是0（A、B同种）或1（A捕食B）。所以w[px]=(w[b]-w[a]+3)%3或w[px]=(1+w[b]-w[a]+3)%3。

查找（路径压缩）

查找中的权值的转移出现于路径压缩中。

借用上图，假设现在的树是x->y->py。我们调用函数find(x)寻找树的根节点。为了改善树的结构，防止出现退化这种极端情况，查询后我们将x直接与根节点相连。但x的权值必须做出相应的改变。

路径压缩前，w[x]相当于从x指向y的向量，路径压缩后，相当于从x指向py的向量。根据向量的加法，我们只需将从x到根节点间的权值累加就得到了新的w[x]。之后再对3取余即可。

代码

在码代码之前，我们再整理一遍思路。

只有在一棵树上的数据才能判断真假。如果在两棵树上就需要先合并，这样才能找到权值间的关系，然后进一步判断与输入数据是否矛盾。

  1. 判断输入不合法的情况，直接continue;

  2. x和y同种

    如果x、y已经在一棵树上，判断w[x]和w[y]是否相等；如果不在，合并后更新权值。

  3. x和y不同种

    如果x、y已经在一棵树上，判断x和y是否为捕食关系((w[x]-w[y]+3)%3是否等于1);如果不在，合并后更新权值。

#include<iostream>

using namespace std;

int par[50050];
int w[50050];
int N, M, d, x, y, ans;

int find(int x)
{
    if(x==par[x])
        return x;
    else
    {
        //保存还没有路径合并时的父节点值
        int n=par[x];
        par[x]=find(par[x]);
        //合并路径后所有子节点都直接指向根，如果前面没有保存父节点的信息就无法更新权值
        w[x]=(w[x]+w[n])%3;
    }
    return par[x];
}

int main(void)
{
    cin>>N>>M;
    for(int i=0; i<=N; i++)
    {
        par[i]=i;
        w[i]=0;
    }
    for(int i=0; i<M; i++)
    {
        cin>>d>>x>>y;
        //排除输入错误
        if(x>N||y>N||(d==2&&x==y))
        {
            ans++;
            continue;
        }
        int fx=find(x);
        int fy=find(y);
        //同种
        if(d==1)
        {
            /*
            我在这里犯了一个卡了自己一晚上的低级错误。我在写多层if-else嵌套时，像写for循环时一样
            省略了内部只有一个语句的if循环的大括号，导致整个逻辑乱掉了。以后引以为戒！！！
            */
            if(fx==fy)
            {
                if(w[x]!=w[y])
                    ans++;
            } 
            else
            {
                par[fx]=fy;
                //加3再取余处理可能出现的负数
                w[fx]=(w[y]-w[x]+3)%3;
            }
        }
        //不同种
        else
        {
            if(fx==fy)
            {
                /*
                这里我也写了一个很迷的bug，如果省略a而直接将表达式放在if种判断似乎并不能得到正确的
                结果，以后注意。
                */
                int a=(w[x]-w[y]+3)%3;
                if(a!=1)
                    ans++;
            }
            else
            {
                par[fx]=fy;
                w[fx]=(w[y]-w[x]+3+1)%3;
            }
        }
    }
    cout<<ans<<endl;
    return 0;
}
