算法学习笔记——并查集

并查集 算法笔记
算法学习笔记
发布日期:   2020-02-05

并查集介绍

并查集是一种用来管理元素分组情况的数据结构。并查集可以高效地进行如下操作。

  1. 查询元素a和元素b是否属于同一组。
  2. 合并元素a和元素b所在的组。

注意：并查集可以进行合并操作，但不能进行分割操作。

并查集的结构

并查集使用树形结构实现，但不是二叉树。基本操作如下：

  1. 初始化

    我们用n个结点来表示n给元素。最开始时没有边。

  2. 合并

    从一个组的根向另一个组的根连边，将两颗树变成一棵树。也就把两个组并为一个组了

  3. 查询

    为了查询两个节点是否属于同一组，我们需要沿着树往上走，来查询包含这个元素的根是谁。如果两个结点走到了同一个根，那么就可以知道它们属于同一组。

并查集实现中的注意点

在树形结构里，如果发生了退化的情况，那么复杂度就会变得很高。在并查集中需要采用如下方法：

  1. 对于每棵树，记录这棵树的高度(rank)。合并时如果两颗树的rank不同，那么从rank小的向rank大的连边。
  2. 通过路径压缩，可以使并查集更加高效。对于每个节点，一旦向上走到了一次根节点，就把这个节点到父亲的边改为直接连向根。(在使用这种简化的方法时，为了简单起见，即使树的高度发生了变化，我们也不修改rank的值)。

并查集的实现

int par[MAX_N]; //记录每个节点的父亲编号
int rank[MAX_N]; //树的高度

//初始化n个元素
void init(int n)
{
    for(int i=0; i<n; i++)
    {
        //开始时没有边，树的高度为0
        rank[i]=0;
        //如果一个节点的父亲是自己，那它就是根节点
        //开始时所有节点都是根节点
        par[i]=i;
    }
}

//查询树的根
int find(int x)
{
    if(par[x]==x)
        return x;
    else
        //如果不是根节点，就继续向上寻找
        //这一步同时完成了路径压缩
        par[x]=find(par[x]);
    return par[x];
}

//合并x和y所属的集合
void unite(int x, int y)
{
    x=find(x);
    y=find(y);
    //如果是同一颗树就不用合并了
    if(x==y)
        return;
    //rank小的向rank大的连边
    if(rank[x]<rank[y])
        par[x]=y;
    else
        par[y]=x;
    //高度相等，合并后高度加1
    if(rank[x]==rank[y])
        rank[x]++;
}

//判断x和y是否属于同一个集合
bool same(int x, int y)
{
    return find(x)==find(y);
}
