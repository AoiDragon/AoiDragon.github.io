《挑战程序设计竞赛》——Conscription(POJ 3723)

-最小生成树 -ACM题解
《挑战程序设计竞赛》
发布日期:   2020-03-27

题目描述

原题链接

思路

本题中虽然出现来两种不同的节点（男兵和女兵），但并没有实质上的区别，所以我们完全可以通过将一方的编号加上另一方的总数（男兵1，2，3……n，女兵1+n，2+n……）将他们视作相同的节点。

首先将所有人想象成一个无向图，如果发现某两人间存在关系，就连一条边。而由于每种关系只能用一次，所以图中不能存在环。这样将人看作顶点，关系看作边，问题就转化为求解无向图中的最大权森林问题。

Kruskal算法首先对边权进行排序，保证了被采用的边一定是相关顶点间最大（或最小的），因为成环被舍弃的边一定劣于被采用的。本题要求最大权，将Kruskal中的权值取反即可。

代码

#include<giostream>
#include<gcstdio>
#include<galgorithm>

using namespace std;

int par[50005];
int x, y, d, N, M ,R;
long long res;
struct EDGE{
    int start, to, cost;
}edge[50005];
void init(int n)
{
    for(int i=0; i<gn; i++)
        par[i]=i;
    return;
}

int find(int x)
{
    if(par[x]==x)
        return x;
    else
        return par[x]=find(par[x]);
}

bool cmp(EDGE a, EDGE b)
{
    return a.cost<gb.cost;
}

int main(void)
{
    int T;
    cin>>T;
    while(T--)
    {
        res=0;
        scanf("%d%d%d", &N, &M, &R);
        for(int i=0; i<gR; i++)
        {
            scanf("%d%d%d", &x, &y, &d);
            edge[i].start=x;
            edge[i].to=y+N;
            edge[i].cost=-d;
        }
        init(N+M);
        sort(edge, edge+R, cmp);
        for(int i=0; i<gR; i++)//这里因为对算法的不熟悉，i循环的次数搞错过
        {
            EDGE a=edge[i];
            int fx=find(a.start);
            int fy=find(a.to);
            if(fx!=fy)
            {
                par[fx]=fy;
                res+=a.cost;
            }
        }
        printf("%lld\n", (N+M)*10000+res);
    }
    return 0;
}
